# 融合智能决策的中小团队任务管理与绩效评估系统——全流程落地实施方案

## 一、 系统核心愿景

本系统旨在为中小规模团队提供一个“轻量化、数据驱动、智能决策”的管理平台。通过将管理者的主观经验（通过 AHP 转化）与客观数据（负载、进度、绩效）相结合，解决任务分配不均、过程监控缺失及绩效评价主观的问题。

## 二、 角色定义与权限体系 (RBAC)

系统基于角色提供功能隔离，确保数据的安全与专业分工。

|角色|核心职能|系统功能权限|
|---|---|---|
|**系统管理员 (Admin)**|系统运维与架构搭建|全局账号管理（增删改查）、团队创建与解散、系统参数配置、日志审计。|
|**团队管理者 (Manager)**|团队决策与资源调度|团队成员审核、**权重决策 (AHP)**、任务指派、全员负载监控、**风险预警响应**、绩效量化定性。|
|**执行成员 (Member)**|任务执行与进度同步|个人任务流转、**进度实时上报 (0-100%)**、个人负载与绩效查看、同事间匿名互评。|

## 三、 系统主要功能模块定义

### 0. 基础保障与身份认证模块

- **登录与安全**：基于 JWT (JSON Web Token) 实现前后端分离的身份验证。支持 Token 自动续期与角色拦截。
    
- **团队管理 (Team CRUD)**：管理者可创建团队，设置团队目标，管理团队公告，归档已完成的项目团队。
    
- **成员管理 (Member CRUD)**：管理员维护用户信息，包括账号状态、基本能力描述。最核心的是**技能标签维护**（如：Java: 5星, 沟通: 4星）。
    

### 1. 团队任务管理与流程配置功能

- **功能定义**：支持任务的全生命周期管理（创建、分发、挂起、完成、归档）。
    
- **流程配置**：记录任务优先级、难度系数、工期。支持设定**任务依赖关系**（如：A任务完成前，B任务不可开始）。
    

### 2. 智能任务分配与优先级决策功能

- **功能定义**：在任务创建后，系统自动计算并推荐“最佳负责人”。
    
- **决策机制**：管理者选定当前关注点（如：更看重能力还是更看重空闲），系统即时反馈推荐列表。
    

### 3. 成员工作负载预测与资源调度功能

- **功能定义**：展示成员未来 1-4 周的预估工作强度。
    
- **调度支持**：通过图表发现“资源过载”节点，支持管理者手动进行任务的快速平移和调配。
    

### 4. 任务执行监控与延期风险预测功能

- **功能定义**：实时扫描所有进行中任务，识别潜在的“烂尾”或“延期”风险。
    
- **预警机制**：在管理者看板中以红/橙/绿三色显示，并计算风险概率。
    

### 5. 智能绩效评估与趋势分析功能

- **功能定义**：周期性（周/月）生成成员绩效报告。
    
- **趋势分析**：通过历史数据分析成员是“稳步提升”还是“状态下滑”，辅助人才培养决策。
    

### 6. 数据可视化与决策支持功能

- **看板内容**：集成 ECharts 展现：任务完成分布饼图、成员负载对比柱状图、风险概率雷达图、绩效得分折线图。
    

## 四、 核心数据结构设计

### 1. 成员实体 (User)

- `id`: 主键。
    
- `skills_json`: 技能向量，如 `{"coding": 0.9, "test": 0.7}`。
    
- `base_capacity`: 每周额定工时（默认 40）。
    
- `avg_performance`: 历史动态平均分。
    

### 2. 任务实体 (Task)

- `id`: 主键。
    
- `difficulty`: 难度系数 ($1.0 \sim 5.0$)。
    
- `priority`: 优先级 ($1 \sim 5$)。
    
- `est_hours`: 预估总耗时。
    
- `deadline`: 截止时间。
    
- `progress`: 当前进度值 ($0 \sim 100$)。
    

### 3. 权重判断矩阵 (AHP_Matrix)

- `matrix_data`: 存储两两对比的原始标度（$1 \sim 9$）。
    
- `final_weights`: 存储计算后的指标权重向量。
    

## 五、 核心算法深度拆解 (无省略)

### 1. 权重确定：AHP 层次分析法

- **算法逻辑**：解决“指标重要性”如何确定的问题。
    
- **实现步骤**：
    
    1. **构造判断矩阵**：管理者对“能力、负载、绩效”进行两两比较。若认为能力比负载重要得多，填入 5；反之填入 1/5。
        
    2. **求特征向量**：对矩阵进行按列归一化，再求各行平均值，得到权重向量 $W = [w_1, w_2, w_3]$。
        
    3. **一致性检验**：计算一致性指标 $CI = \frac{\lambda_{max} - n}{n - 1}$ 和比例 $CR = \frac{CI}{RI}$。若 $CR < 0.1$，权重有效，否则要求重填。
        
- **作用**：让系统理解当前管理者的偏好。
    

### 2. 任务分配：多指标加权评分

- **算法逻辑**：计算成员与任务的匹配总分。
    
- **公式**：$Score_{i} = w_{skill} \cdot S_{i} + w_{load} \cdot (1 - L_{i}) + w_{perf} \cdot P_{i}$。
    
    - $S_i$：成员技能与任务需求的匹配分。
        
    - $L_i$：当前成员已分配任务的总工时 / 额定容量（负载率）。
        
    - $P_i$：历史绩效分。
        
- **作用**：自动从高到低排序，推荐执行人。
    

### 3. 负载预测：指数平滑法 (Exponential Smoothing)

- **算法逻辑**：预测未来的工作强度。
    
- **公式**：$F_{t+1} = \alpha \cdot Y_{t} + (1 - \alpha) \cdot F_{t}$。
    
    - $Y_t$：本周实际投入工时。
        
    - $F_t$：本周预测值。
        
    - $\alpha$：平滑系数（通常取 $0.3 \sim 0.5$）。
        
- **作用**：结合已分配的未来任务，预测后续周期的成员忙闲度。
    

### 4. 风险预测：基于逻辑回归 (Logistic Regression) 的预警

- **算法逻辑**：计算任务延期的概率。
    
- **输入特征**：$x_1$(进度滞后率), $x_2$(任务难度), $x_3$(执行人延期历史)。
    
    - 进度滞后率 = $(\text{已过时间比} - \text{当前进度比})$。
        
- **公式**：$P = \frac{1}{1 + e^{-(z)}}$，其中 $z = \beta_0 + \beta_1 x_1 + \beta_2 x_2 + \beta_3 x_3$。
    
- **作用**：当 $P > 0.7$ 时，系统自动在看板中将该任务标记为“高风险红色”。
    

### 5. 绩效评价：模糊综合评价法 (FCE)

- **算法逻辑**：处理评价中“含糊”的部分（如：什么是“表现良好”？）。
    
- **实现步骤**：
    
    1. **确定评语集**：$\{优秀, 良好, 合格, 不合格\}$。
        
    2. **构建隶属度矩阵** $R$：由管理者打分、系统数据、互评数据组成。
        
    3. **模糊运算**：$B = W \circ R$（权重向量与隶属度矩阵合成）。
        
    4. **计算综合分**：将 $B$ 向量映射到百分制区间。
        
- **作用**：得出客观、公允的最终绩效分。
    

## 六、 数据流向与交互

1. **认证流**：用户提交账号密码 $\rightarrow$ 后端返回 JWT $\rightarrow$ 前端存储并在后续 API Header 中携带。
    
2. **任务流**：管理者建单 $\rightarrow$ 调用 **AHP 权重** $\rightarrow$ 调用 **匹配模型** $\rightarrow$ 返回推荐人选 $\rightarrow$ 成员接收并开始工作。
    
3. **监控流**：成员每日上报进度 $\rightarrow$ 触发 **风险预测函数** $\rightarrow$ 更新数据库风险状态 $\rightarrow$ 前端可视化看板刷新。
    
4. **绩效流**：任务完成 $\rightarrow$ 采集工时、质量、互评 $\rightarrow$ 触发 **模糊评价算法** $\rightarrow$ 更新 `avg_performance`。
    

## 七、 实施步骤建议

1. **第一阶段** ：搭建基础框架，实现 **User/Team/Task** 的 CRUD 接口及登录拦截。
    
2. **第二阶段**：开发 **AHP 指标配置**和**智能分配**模块，完成最核心的算法集成。
    
3. **第三阶段** ：开发进度追踪与**风险预警逻辑**，集成 **ECharts** 看板展示。
    
4. **第四阶段**：完善**绩效评估系统**，进行系统联调与性能优化。
