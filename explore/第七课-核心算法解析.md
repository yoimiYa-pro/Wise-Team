## 二、AHP 层次分析法

### 2.1 什么是 AHP？

AHP（Analytic Hierarchy Process）层次分析法是一种**"把主观判断变成客观数字"**的方法。

用大白话说：
```
领导想：能力、负载、绩效，哪个更重要？

领导说："能力比负载重要一些，比绩效重要很多"

AHP 把这些"感觉"变成具体的数字：
- 能力权重：0.5
- 负载权重：0.3
- 绩效权重：0.2
```

### 2.2 AHP 的步骤

```
步骤1：构造判断矩阵

        能力    负载    绩效
能力     1       3       5
负载    1/3      1       2
绩效    1/5     1/2      1

含义：
- 能力 vs 负载 = 3：能力比负载"稍微重要"
- 能力 vs 绩效 = 5：能力比绩效"明显重要"
- 负载 vs 绩效 = 2：负载比绩效"稍微重要"

步骤2：计算权重
按列求和 → 归一化 → 按行求平均 → 得到权重

步骤3：一致性检验
检查判断是否"自相矛盾"
```

### 2.3 数字含义表

| 数值 | 含义 |
|------|------|
| 1 | 同等重要 |
| 3 | 稍微重要 |
| 5 | 明显重要 |
| 7 | 强烈重要 |
| 9 | 极端重要 |
| 2,4,6,8 | 中间值 |
| 倒数 | 反向比较 |

### 2.4 代码解析

打开 `algo/AhpCalculator.java`：

```java
public final class AhpCalculator {

    // RI（Random Index）：随机一致性指标，用于检验
    private static final double[] RI = {0, 0, 0.58, 0.9, 1.12, 1.24, 1.32, 1.41, 1.45, 1.49};

    // 结果类：包含权重和其他计算结果
    @Data
    public static class AhpResult {
        private double[] weights;    // 最终权重
        private double lambdaMax;    // 最大特征值
        private double ci;           // 一致性指标
        private double cr;           // 一致性比率
        private boolean consistent;  // 是否通过一致性检验
    }

    public static AhpResult compute(double[][] matrix) {
        int n = matrix.length;  // 矩阵维度（3×3）
        
        // ========== 步骤1：按列归一化 ==========
        double[][] norm = new double[n][n];
        for (int j = 0; j < n; j++) {
            // 计算第 j 列的和
            double colSum = 0;
            for (int i = 0; i < n; i++) {
                colSum += matrix[i][j];
            }
            // 归一化：每个元素除以列和
            for (int i = 0; i < n; i++) {
                norm[i][j] = matrix[i][j] / colSum;
            }
        }
        
        // ========== 步骤2：按行求平均得到权重 ==========
        double[] w = new double[n];
        for (int i = 0; i < n; i++) {
            double rowSum = 0;
            for (int j = 0; j < n; j++) {
                rowSum += norm[i][j];
            }
            w[i] = rowSum / n;  // 每行的平均值就是权重
        }
        
        // ========== 步骤3：计算最大特征值 ==========
        double lambdaMax = 0;
        for (int i = 0; i < n; i++) {
            double sum = 0;
            for (int j = 0; j < n; j++) {
                sum += matrix[i][j] * w[j];
            }
            lambdaMax += sum / w[i];
        }
        lambdaMax /= n;
        
        // ========== 步骤4：一致性检验 ==========
        // CI = (λmax - n) / (n - 1)
        double ci = (lambdaMax - n) / (n - 1);
        // CR = CI / RI
        double ri = n < RI.length ? RI[n] : RI[RI.length - 1];
        double cr = ri > 0 ? ci / ri : 0;
        
        // CR < 0.1 表示通过一致性检验
        boolean consistent = cr < 0.1;
        
        // 封装结果
        AhpResult r = new AhpResult();
        r.setWeights(w);
        r.setLambdaMax(lambdaMax);
        r.setCi(ci);
        r.setCr(cr);
        r.setConsistent(consistent);
        return r;
    }
}
```

### 2.5 计算示例

假设判断矩阵：
```
      能力   负载   绩效
能力   1     3      5
负载  1/3    1      2
绩效  1/5   1/2     1
```

计算过程：

```
步骤1：按列求和
第1列：1 + 1/3 + 1/5 = 1.533
第2列：3 + 1 + 1/2 = 4.5
第3列：5 + 2 + 1 = 8

步骤2：按列归一化
        能力      负载      绩效
能力  1/1.533=0.652   3/4.5=0.667   5/8=0.625
负载  0.333/1.533=0.217  1/4.5=0.222   2/8=0.25
绩效  0.2/1.533=0.130   0.5/4.5=0.111  1/8=0.125

步骤3：按行求平均
w[能力] = (0.652 + 0.667 + 0.625) / 3 = 0.648
w[负载] = (0.217 + 0.222 + 0.25) / 3 = 0.230
w[绩效] = (0.130 + 0.111 + 0.125) / 3 = 0.122

最终权重：能力=0.65, 负载=0.23, 绩效=0.12
```

## 三、技能匹配算法

### 3.1 什么是技能匹配？

```
任务要求：Java、MySQL、前端开发
用户技能：Java(0.9)、MySQL(0.7)、Python(0.5)

匹配度 = ?
```

### 3.2 余弦相似度

用余弦相似度计算两个"向量"的相似程度：

```
把技能看作向量：

任务要求 = [Java, MySQL, 前端]
        = [1.0,   1.0,   1.0]    （要求有的技能为1）

用户技能 = [Java, MySQL, Python, 前端]
        = [0.9,   0.7,   0.5,    0]

余弦相似度 = cos(θ) = (A·B) / (|A| × |B|)
```

### 3.3 代码解析

打开 `algo/SkillMatch.java`：

```java
public final class SkillMatch {

    // 计算余弦相似度
    public static double cosineSimilarity(String requiredJson, String userSkillsJson) {
        // 解析 JSON 为 Map
        Map<String, Double> req = parse(requiredJson);    // 任务要求的技能
        Map<String, Double> usr = parse(userSkillsJson);  // 用户拥有的技能
        
        // 如果任务没有技能要求，默认返回 0.85
        if (req.isEmpty()) {
            return 0.85;
        }
        
        // 合并所有技能名称
        Set<String> keys = new HashSet<>(req.keySet());
        keys.addAll(usr.keySet());
        
        // 计算点积和模长
        double dot = 0, n1 = 0, n2 = 0;
        for (String k : keys) {
            double a = req.getOrDefault(k, 0.0);  // 任务要求的值
            double b = usr.getOrDefault(k, 0.0);  // 用户拥有的值
            dot += a * b;      // 点积：A·B
            n1 += a * a;       // A 的模长的平方
            n2 += b * b;       // B 的模长的平方
        }
        
        // 避免除零
        if (n1 <= 0 || n2 <= 0) {
            return 0;
        }
        
        // 余弦相似度 = 点积 / (模长相乘)
        return Math.min(1.0, dot / (Math.sqrt(n1) * Math.sqrt(n2)));
    }
    
    // 解析 JSON
    private static Map<String, Double> parse(String json) {
        // 把 JSON 字符串转换成 Map
        // 例如：{"java": 0.9, "mysql": 0.7} → Map{"java"=0.9, "mysql"=0.7}
    }
}
```

### 3.4 计算示例

```
任务要求：{"java": 1.0, "mysql": 1.0}
用户技能：{"java": 0.9, "mysql": 0.7, "python": 0.5}

计算过程：
合并技能：{java, mysql, python}

技能值：
        java    mysql   python
任务：   1.0     1.0     0.0
用户：   0.9     0.7     0.5

点积 = 1.0×0.9 + 1.0×0.7 + 0.0×0.5 = 1.6

|任务| = √(1.0² + 1.0² + 0.0²) = √2 = 1.414
|用户| = √(0.9² + 0.7² + 0.5²) = √1.55 = 1.245

余弦相似度 = 1.6 / (1.414 × 1.245) = 1.6 / 1.761 = 0.908

匹配度 = 0.908（很高！）
```

## 四、任务分配算法

### 4.1 综合评分公式

任务分配综合考虑三个因素：
- 技能匹配度（会不会做）
- 空闲程度（有没有时间）
- 历史绩效（做得好不好）

```
总分 = w₁ × 技能匹配 + w₂ × (1 - 负载率) + w₃ × 绩效分

其中：
- w₁, w₂, w₃ 是 AHP 计算出的权重
- 负载率 = 已分配工时 / 额定容量
- (1 - 负载率) = 空闲程度
```

### 4.2 代码解析

打开 `service/AssignmentService.java`：

```java
public List<Map<String, Object>> recommend(Long teamId, String requiredSkillsJson, UserPrincipal actor) {
    
    // 获取 AHP 权重
    double[] w = ahpService.weightsOrDefault(teamId);
    // w[0] = 技能权重
    // w[1] = 负载权重
    // w[2] = 绩效权重
    
    // 获取团队成员
    List<Long> userIds = teamMemberMapper.findApprovedUserIds(teamId);
    
    List<Map<String, Object>> out = new ArrayList<>();
    
    for (Long uid : userIds) {
        User u = userMapper.findById(uid);
        
        // 1. 计算技能匹配分
        double s = SkillMatch.cosineSimilarity(requiredSkillsJson, u.getSkillsJson());
        
        // 2. 计算负载率
        BigDecimal rem = taskMapper.sumRemainingHoursByAssignee(uid);  // 已分配工时
        BigDecimal cap = u.getBaseCapacity();  // 额定容量（默认40小时）
        double load = rem.divide(cap, 4, RoundingMode.HALF_UP).doubleValue();
        load = Math.min(1.0, Math.max(0, load));  // 限制在 0-1
        
        // 3. 计算绩效分（归一化到 0-1）
        double perf = u.getAvgPerformance().doubleValue() / 100.0;
        
        // 4. 计算总分
        double score = w[0] * s + w[1] * (1 - load) + w[2] * perf;
        
        out.add(Map.of(
            "userId", uid,
            "totalScore", score
        ));
    }
    
    // 按总分从高到低排序
    out.sort(Comparator.comparingDouble(m -> -((Number) m.get("totalScore")).doubleValue()));
    
    return out;
}
```

假设权重：w = [0.5, 0.3, 0.2]

|------|----------|--------|--------|--------|------|

## 五、风险预测算法

### 5.1 逻辑回归模型

用逻辑回归预测"任务是否会延期"的概率：

```
延期概率 P = 1 / (1 + e^(-z))

其中：
z = β₀ + β₁×进度滞后率 + β₂×任务难度 + β₃×延期历史

进度滞后率 = 已过时间比例 - 当前进度比例
```

### 5.2 代码解析

打开 `algo/RiskModel.java`：

```java
public final class RiskModel {

    // 计算延期概率
    public static BigDecimal delayProbability(
            AppProperties.Risk risk,    // 配置参数
            double progressPct,          // 当前进度（0-100）
            LocalDate start,             // 开始日期
            LocalDate deadline,          // 截止日期
            double difficultyOneToFive,  // 难度（1-5）
            double delayHistory01        // 延期历史（0-1）
    ) {
        // 1. 计算时间比例（已过时间 / 总时间）
        double timeRatio;
        if (deadline != null && start != null) {
            long total = ChronoUnit.DAYS.between(start, deadline);    // 总天数
            long elapsed = ChronoUnit.DAYS.between(start, LocalDate.now());  // 已过天数
            timeRatio = Math.min(1.0, elapsed / (double) total);
        } else {
            timeRatio = 0.5;  // 默认一半
        }
        
        // 2. 计算进度比例
        double progRatio = progressPct / 100.0;
        
        // 3. 计算特征值
        double x1 = timeRatio - progRatio;  // 进度滞后率
        double x2 = difficultyOneToFive / 5.0;  // 难度归一化
        double x3 = delayHistory01;  // 延期历史
        
        // 4. 计算 z 值
        double z = risk.getBeta0() 
                 + risk.getBeta1() * x1 
                 + risk.getBeta2() * x2 
                 + risk.getBeta3() * x3;
        
        // 5. 计算 sigmoid 值（0-1 之间的概率）
        double p = 1.0 / (1.0 + Math.exp(-z));
        
        return BigDecimal.valueOf(p);
    }
    
    // 根据概率确定风险等级
    public static String level(AppProperties.Risk risk, BigDecimal p) {
        double v = p.doubleValue();
        if (v >= risk.getRedThreshold()) {    // 默认 0.7
            return "RED";
        }
        if (v >= risk.getOrangeThreshold()) {  // 默认 0.4
            return "ORANGE";
        }
        return "GREEN";
    }
}
```

### 5.3 Sigmoid 函数图解

```
P = 1 / (1 + e^(-z))

        1.0 ┤                    ╭──────────
            │                  ╭╯
        0.5 ┤─────────────────╯
            │              ╭╯
        0.0 ┤──────────────╯────────────────
           -4   -2    0    2    4    z值

当 z > 0 时，P > 0.5
当 z < 0 时，P < 0.5
z 越大，P 越接近 1
z 越小，P 越接近 0
```

### 5.4 计算示例

```
任务情况：
- 当前进度：30%
- 开始日期：2024-01-01
- 截止日期：2024-01-31（还有 10 天）
- 难度：4
- 执行人延期历史：0.3

计算：
时间比例 = 21/31 = 0.68
进度比例 = 0.30
进度滞后率 = 0.68 - 0.30 = 0.38

假设参数：
β₀ = -2, β₁ = 3, β₂ = 1, β₃ = 2

z = -2 + 3×0.38 + 1×0.8 + 2×0.3
  = -2 + 1.14 + 0.8 + 0.6
  = 0.54

P = 1 / (1 + e^(-0.54))
  = 1 / (1 + 0.583)
  = 0.632

风险等级：0.632 > 0.4 → 橙色（中等风险）
```

## 六、绩效评估算法

### 6.1 模糊综合评价法

绩效评估考虑三个方面：
- 管理者评价
- 系统数据
- 同事互评

用模糊综合评价法把"模糊的评价"变成"具体的分数"。

### 6.2 评语等级

```
4 个评语等级：优秀、良好、合格、不合格
对应分数：    95、  82、   68、   50
```

### 6.3 代码解析

打开 `algo/FuzzyComprehensiveEvaluation.java`：

```java
public final class FuzzyComprehensiveEvaluation {

    // 四个评语等级对应的分数
    private static final double[] GRADE_SCORES = {95, 82, 68, 50};

    /**
     * @param r 隶属度矩阵（3×4）
     *          行：管理者评价、系统数据、同事互评
     *          列：优秀、良好、合格、不合格
     * @param w 权重向量（长度3），表示三方面的权重
     */
    public static double score(double[][] r, double[] w) {
        // 合成运算：B = W × R
        double[] b = new double[4];
        for (int j = 0; j < 4; j++) {
            double s = 0;
            for (int i = 0; i < 3; i++) {
                s += w[i] * r[i][j];
            }
            b[j] = s;
        }
        
        // 归一化
        double sumB = Arrays.stream(b).sum();
        for (int j = 0; j < 4; j++) {
            b[j] /= sumB;
        }
        
        // 加权平均计算最终分数
        double out = 0;
        for (int j = 0; j < 4; j++) {
            out += b[j] * GRADE_SCORES[j];
        }
        
        return Math.max(0, Math.min(100, out));
    }
}
```

### 6.4 计算示例

```
输入数据：
权重 w = [0.4, 0.35, 0.25]（管理者、系统、互评）

隶属度矩阵 R：
              优秀   良好   合格   不合格
管理者评价：   0.6    0.3    0.1    0.0
系统数据：     0.4    0.4    0.2    0.0
同事互评：     0.5    0.3    0.2    0.0

计算 B = W × R：
B[优秀] = 0.4×0.6 + 0.35×0.4 + 0.25×0.5 = 0.505
B[良好] = 0.4×0.3 + 0.35×0.4 + 0.25×0.3 = 0.335
B[合格] = 0.4×0.1 + 0.35×0.2 + 0.25×0.2 = 0.160
B[不合格] = 0.4×0 + 0.35×0 + 0.25×0 = 0

归一化（总和=1.0）：
B = [0.505, 0.335, 0.160, 0]

计算最终分数：
分数 = 0.505×95 + 0.335×82 + 0.160×68 + 0×50
     = 47.975 + 27.47 + 10.88 + 0
     = 86.325

最终绩效分：86.33（良好偏上）
```

## 七、指数平滑预测算法

### 7.1 什么是指数平滑？

指数平滑用于**预测未来的工作负载**：

```
核心思想：最近的数据更重要

公式：F(t+1) = α × Y(t) + (1-α) × F(t)

其中：
- F(t+1)：下一期预测值
- Y(t)：本期实际值
- F(t)：本期预测值
- α：平滑系数（0-1），越大越重视近期数据
```

### 7.2 代码解析

打开 `algo/ExponentialSmoothing.java`：

```java
public final class ExponentialSmoothing {

    /**
     * @param historyOldestFirst 历史数据（从旧到新）
     * @param alpha 平滑系数（0-1）
     * @param initialF 初始预测值
     * @return 下一期预测值
     */
    public static double nextForecast(
            List<Double> historyOldestFirst, 
            double alpha, 
            double initialF) {
        
        if (historyOldestFirst == null || historyOldestFirst.isEmpty()) {
            return initialF;
        }
        
        double f = initialF;  // 初始预测值
        
        // 逐期更新预测
        for (double y : historyOldestFirst) {
            f = alpha * y + (1 - alpha) * f;
        }
        
        return f;
    }
}
```

### 7.3 计算示例

```
历史数据（最近4周工时）：[32, 38, 35, 40]
平滑系数 α = 0.4
初始预测 F = 35

逐步计算：
第1周后：F = 0.4 × 32 + 0.6 × 35 = 33.8
第2周后：F = 0.4 × 38 + 0.6 × 33.8 = 35.48
第3周后：F = 0.4 × 35 + 0.6 × 35.48 = 35.29
第4周后：F = 0.4 × 40 + 0.6 × 35.29 = 37.17

预测下周工时：约 37.2 小时
```

## 八、算法流程总图

### 8.1 任务分配流程

```
┌─────────────────────────────────────────────────────────┐
│  管理者创建任务                                           │
│  填写：任务要求、所需技能、难度、截止日期...               │
└─────────────────────────────────────────────────────────┘
                        ↓
┌─────────────────────────────────────────────────────────┐
│  系统调用 AHP 获取权重                                    │
│  权重 = [技能权重, 负载权重, 绩效权重]                    │
└─────────────────────────────────────────────────────────┘
                        ↓
┌─────────────────────────────────────────────────────────┐
│  遍历每个团队成员                                         │
│  1. 计算技能匹配分（SkillMatch.cosineSimilarity）         │
│  2. 查询当前负载率                                        │
│  3. 获取历史绩效分                                        │
│  4. 计算总分 = w₁×技能 + w₂×空闲 + w₃×绩效               │
└─────────────────────────────────────────────────────────┘
                        ↓
┌─────────────────────────────────────────────────────────┐
│  按总分排序，返回推荐列表                                  │
│  [张三: 0.85, 李四: 0.72, 王五: 0.65]                    │
└─────────────────────────────────────────────────────────┘
                        ↓
┌─────────────────────────────────────────────────────────┐
│  管理者看到推荐列表，选择最合适的人                        │
└─────────────────────────────────────────────────────────┘
```

                        ↓
         ┌──────────────┴──────────────┐
         ↓                              ↓
