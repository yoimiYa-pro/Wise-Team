# 第七章 总结与展望

## 7.1 工作总结

本文基于 **Spring Boot** 与 **Vue** 技术，设计并实现了一个**智能团队任务管理与绩效评估系统**（Wise-Team）。该系统以 **B/S、前后端分离** 架构为基础，旨在为中小规模团队提供任务协作、负载感知与绩效闭环的一体化支撑，以满足团队管理者与普通成员在「指派—执行—评价」链路中的日常使用需求。系统主要包含用户认证与权限、系统管理与审计、团队与成员治理、团队任务协作（含依赖、状态审批、风险预警与智能指派）、周工时采集与指数平滑负荷预测、基于 **AHP** 的指派权重配置、绩效周期与同事互评及**模糊综合评价**关账、站内消息、技能词典与管理者看板等核心模块，并辅以 **Ant Design Vue** 与 **ECharts** 构建界面与可视化，注重菜单权限与团队语境下的操作连贯性。

在技术实现方面，本系统采用 **Maven** 进行后端项目管理；服务端基于 **Spring Boot** 搭建 REST 接口层，通过 **Spring Security** 与 **JWT（访问令牌 + 刷新令牌）** 实现无会话鉴权与令牌刷新，密码等敏感字段不落明文传输对比依赖摘要校验；持久层采用 **MyBatis** 访问 **MySQL**，数据库结构由 **Flyway** 版本化迁移维护。前端采用 **Vue 3、TypeScript** 与 **Vite** 构建工程化单页应用，使用 **Axios** 统一发起请求并在拦截器中附带 Bearer 令牌、处理 401 与静默刷新。开发过程遵循需求分析、总体设计、模块实现与系统测试等基本软件工程步骤，与第三至第六章论述相一致。

通过系统测试（含第六章所列功能用例、算法核对及非功能抽查），验证了系统主要功能与前期需求的一致性；并对核心接口的响应表现、基于角色的访问控制及主流浏览器兼容性进行了综合评估。测试结果表明，在本地与类演示环境下系统响应可接受，越权访问可被拦截，在 **Chrome / Edge** 等主流浏览器中可完成主要业务流程，体现了较好的可用性与适用性。

## 7.2 不足与未来展望

本系统的设计与实现，将**层次分析法、模糊综合评价、指数平滑与风险规则**与团队管理场景相结合，有助于提升任务指派的透明度与绩效评价的可解释性，并为后续扩展（更多数据源、更丰富协作形态）预留了接口与配置层面的空间。受本科毕业设计周期与篇幅所限，自动化测试覆盖、生产级压测与高可用部署等内容尚未充分展开；算法参数亦多在示例数据下调试，仍有在实际团队中持续校准的余地。

为进一步提升系统价值与用户体验，下一步工作计划包括：

**(1) 引入更多数据驱动的优化手段。** 在保护隐私与脱敏的前提下，积累工时完成、任务延期与互评结果等统计特征，对指数平滑系数、风险阈值与指派排序权重做标定或对比实验；在可解释性约束下探索轻量级推荐与预警改进，使「智能」模块逐步从规则配置过渡到**有据可依的参数演化**。

**(2) 扩展协作形态与外部联通。** 在现有站内消息基础上，考虑任务级评论、附件或简易文档链接，增强执行过程的可追溯性；条件成熟时探索与企业即时通讯、邮件或待办系统的对接，减少信息孤岛。

**(3) 界面与交互的持续改进。** 在前端技术演进与组件库更新背景下，统一空态、加载与错误反馈，优化团队选择、任务批量操作等高频路径；在保障功能完备的前提下，提升视觉层次与操作直觉，降低管理者与成员的学习成本。

**(4) 探索多端与移动化应用。** 在响应式布局基础上，评估独立移动端页面或小程序形态，使管理者能够外出查阅看板与审批待办、成员能够移动端上报工时与查看任务，满足随时随地协作的需求。

## 本章小结

本章总结了 Wise-Team 的总体目标、技术路线与测试结论，说明了系统在毕设范围内的局限，并从数据驱动优化、协作扩展、界面体验与多端应用四个方面给出了后续工作计划，与全文的研究目标与技术选型相呼应。
