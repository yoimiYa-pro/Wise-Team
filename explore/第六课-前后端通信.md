# 第六课：前后端通信 —— API 调用详解

## 一、什么是前后端通信？

### 1.1 用大白话说

想象你在餐厅点餐：

```
你（前端） → 服务员（API） → 厨房（后端）

1. 你看菜单，告诉服务员想要什么
2. 服务员把订单送到厨房
3. 厨房做好菜
4. 服务员把菜端给你
```

在 Web 开发中：
- **前端** = 你（发请求、收响应）
- **API** = 服务员（传递信息）
- **后端** = 厨房（处理请求、返回结果）

### 1.2 HTTP 请求基础

HTTP 是前端和后端"对话"的语言：

```
请求（Request）—— 前端发给后端：
┌─────────────────────────────────────────────────────────┐
│  请求方法：GET / POST / PUT / DELETE                      │
│  请求地址：/api/tasks                                    │
│  请求头：Content-Type: application/json                  │
│  请求体：{ "title": "新任务", "deadline": "2024-01-20" }  │
└─────────────────────────────────────────────────────────┘

响应（Response）—— 后端返回前端：
┌─────────────────────────────────────────────────────────┐
│  状态码：200（成功）/ 400（错误）/ 401（未登录）...        │
│  响应头：Content-Type: application/json                  │
│  响应体：{ "id": 1, "title": "新任务", "status": "CREATED" }│
└─────────────────────────────────────────────────────────┘
```

### 1.3 常用 HTTP 方法

| 方法 | 用途 | 比喻 |
|------|------|------|
| GET | 获取数据 | 查看菜单 |
| POST | 创建数据 | 下单 |
| PUT | 更新数据 | 改单 |
| DELETE | 删除数据 | 取消订单 |

### 1.4 HTTP 状态码

| 状态码 | 含义 | 说明 |
|--------|------|------|
| 200 | 成功 | 请求处理成功 |
| 201 | 已创建 | 成功创建了新资源 |
| 400 | 请求错误 | 参数不对 |
| 401 | 未授权 | 没登录或 token 过期 |
| 403 | 禁止访问 | 没权限 |
| 404 | 未找到 | 资源不存在 |
| 500 | 服务器错误 | 后端出问题了 |

## 二、Axios 是什么？

### 2.1 为什么用 Axios？

原生 JavaScript 发请求很麻烦：

```javascript
// 原生方式（繁琐）
const xhr = new XMLHttpRequest();
xhr.open('GET', '/api/tasks');
xhr.setRequestHeader('Authorization', 'Bearer ' + token);
xhr.onload = function() {
  if (xhr.status === 200) {
    const data = JSON.parse(xhr.responseText);
    console.log(data);
  }
};
xhr.send();
```

用 Axios 简单很多：

```javascript
// Axios 方式（简洁）
const { data } = await axios.get('/api/tasks', {
  headers: { Authorization: 'Bearer ' + token }
});
console.log(data);
```

### 2.2 Axios 的优点

- **简洁的 API**：`get`、`post`、`put`、`delete` 方法
- **自动转换 JSON**：不用手动解析
- **拦截器**：可以在请求前后做统一处理
- **错误处理**：统一处理错误
- **取消请求**：可以取消正在进行的请求

## 三、项目的 API 客户端

### 3.1 创建 Axios 实例

打开 `frontend/src/api/client.ts`：

```typescript
import axios from "axios";

// 创建一个 Axios 实例
const client = axios.create({ baseURL: "/api" });
// baseURL：所有请求的基础地址
// 比如 client.get('/tasks') 实际请求的是 /api/tasks

export default client;
```

为什么创建实例而不是直接用 `axios`？

```
直接用 axios：
axios.get('http://localhost:8080/api/tasks')

用实例：
client.get('/tasks')

优点：
1. 统一配置基础地址，改一个地方就行
2. 可以配置拦截器，统一处理请求和响应
3. 多个实例可以有不同的配置
```

### 3.2 请求拦截器

```typescript
// 请求拦截器：在每个请求发送前执行
client.interceptors.request.use((config) => {
  // 从 localStorage 获取访问令牌
  const access = localStorage.getItem("accessToken");
  
  // 如果存在令牌，添加到请求头
  if (access) {
    config.headers.Authorization = `Bearer ${access}`;
  }
  
  return config;
});
```

执行流程：

```
调用 client.get('/tasks')
        ↓
请求拦截器执行
        ↓
检查 localStorage 中有没有 accessToken
        ↓
有：添加到请求头
Authorization: Bearer eyJhbGciOiJIUzI1NiJ9...
        ↓
发送请求到后端
GET /api/tasks
Headers: { Authorization: 'Bearer ...' }
```

### 3.3 响应拦截器

```typescript
// 响应拦截器：在每个响应返回后执行
client.interceptors.response.use(
  (r) => r,  // 成功的响应，直接返回
  async (err) => {
    const original = err.config;  // 原始请求配置
    
    // 如果是 401 错误（未授权）
    if (err.response?.status === 401 && !(original._retry)) {
      original._retry = true;  // 标记为已重试
      
      const rt = localStorage.getItem("refreshToken");
      if (!rt) {
        // 没有刷新令牌，跳转登录页
        localStorage.clear();
        window.location.href = "/login";
        return Promise.reject(err);
      }
      
      try {
        // 用刷新令牌获取新的访问令牌
        const { data } = await axios.post("/api/auth/refresh", {
          refreshToken: rt,
        });
        
        // 保存新令牌
        localStorage.setItem("accessToken", data.accessToken);
        localStorage.setItem("refreshToken", data.refreshToken);
        
        // 用新令牌重新发送原请求
        original.headers.Authorization = `Bearer ${data.accessToken}`;
        return client(original);
      } catch {
        // 刷新失败，跳转登录页
        localStorage.clear();
        window.location.href = "/login";
      }
    }
    
    return Promise.reject(err);
  }
);
```

令牌刷新流程：

```
用户请求 /api/tasks
        ↓
令牌过期，后端返回 401
        ↓
响应拦截器捕获 401 错误
        ↓
用 refreshToken 调用 /api/auth/refresh
        ↓
获取新的 accessToken 和 refreshToken
        ↓
用新令牌重新发送原请求
        ↓
成功返回数据
```

## 四、实际使用示例

### 4.1 登录请求

```typescript
// 在 LoginView.vue 中
import client from "../api/client";

async function onFinish() {
  try {
    // POST 请求，发送用户名密码
    const { data } = await client.post("/auth/login", {
      username: "admin",
      password: "Admin@123"
    });
    
    // data 包含：
    // {
    //   accessToken: "eyJhbGci...",
    //   refreshToken: "eyJhbGci...",
    //   role: "ADMIN",
    //   userId: 1,
    //   username: "admin"
    // }
    
    // 保存到 localStorage
    localStorage.setItem("accessToken", data.accessToken);
    localStorage.setItem("refreshToken", data.refreshToken);
    localStorage.setItem("role", data.role);
    
  } catch (error) {
    // 请求失败处理
    message.error("登录失败");
  }
}
```

### 4.2 获取任务列表

```typescript
// GET 请求
const { data } = await client.get("/tasks", {
  params: { teamId: 1 }  // 查询参数
});

// 等价于 GET /api/tasks?teamId=1
```

### 4.3 创建任务

```typescript
// POST 请求
const { data } = await client.post("/tasks", {
  title: "开发登录功能",
  description: "实现用户登录模块",
  difficulty: 3,
  priority: 2,
  deadline: "2024-02-01",
  assigneeId: 5
});
```

### 4.4 更新任务

```typescript
// PUT 请求
const { data } = await client.put("/tasks/123", {
  title: "新标题",
  progress: 50
});

// 或者 PATCH 请求（部分更新）
const { data } = await client.patch("/tasks/123", {
  progress: 60
});
```

### 4.5 删除任务

```typescript
// DELETE 请求
await client.delete("/tasks/123");
```

## 六、错误处理

### 6.1 统一错误处理

```typescript
async function fetchData() {
  try {
    const { data } = await client.get("/tasks/123");
    return data;
  } catch (error) {
    if (axios.isAxiosError(error)) {
      // Axios 错误
      if (error.response) {
        // 服务器返回了错误响应
        switch (error.response.status) {
          case 400:
            message.error("请求参数错误");
            break;
          case 401:
            message.error("请先登录");
            break;
          case 403:
            message.error("没有权限");
            break;
          case 404:
            message.error("资源不存在");
            break;
          case 500:
            message.error("服务器错误");
            break;
          default:
            message.error("请求失败");
        }
      } else if (error.request) {
        // 请求已发送但没有收到响应
        message.error("网络错误，请检查网络连接");
      }
    } else {
      // 其他错误
      message.error("发生未知错误");
    }
    throw error;
  }
}
```

### 6.2 后端错误响应格式

后端返回的错误格式：

```json
{
  "error": "用户名已存在"
}
```

在拦截器中统一处理：

```typescript
client.interceptors.response.use(
  (r) => r,
  (err) => {
    if (err.response?.data?.error) {
      message.error(err.response.data.error);
    }
    return Promise.reject(err);
  }
);
```

## 七、跨域问题（CORS）

### 7.1 什么是跨域？

```
前端运行在：http://localhost:5173
后端运行在：http://localhost:8080

前端请求后端，就是"跨域"。

浏览器会阻止跨域请求，除非后端允许。
```

### 7.2 解决方案

**方案一：后端配置 CORS**

在后端添加配置，允许前端访问：

```java
@Configuration
public class WebConfig implements WebMvcConfigurer {
    @Override
    public void addCorsMappings(CorsRegistry registry) {
        registry.addMapping("/api/**")
            .allowedOrigins("http://localhost:5173")
            .allowedMethods("GET", "POST", "PUT", "DELETE")
            .allowCredentials(true);
    }
}
```

**方案二：前端代理**

在 Vite 配置中添加代理：

```typescript
// vite.config.ts
export default {
  server: {
    proxy: {
      '/api': {
        target: 'http://localhost:8080',
        changeOrigin: true
      }
    }
  }
}
```

这样前端请求 `/api/tasks` 会被代理到 `http://localhost:8080/api/tasks`。

// 在拦截器中打印请求信息
client.interceptors.request.use((config) => {

  console.log('请求头:', config.headers);
  console.log('请求体:', config.data);

// 打印响应信息
client.interceptors.response.use((response) => {
  console.log('响应状态:', response.status);
