# 第四课：业务逻辑层 —— 业务是怎么处理的

## 一、什么是业务逻辑？

### 1.1 用大白话说

业务逻辑就是**"做事的规则"**：

```
现实生活中的业务逻辑：

去银行取钱：
1. 检查你的卡是不是真的
2. 检查密码对不对
3. 检查余额够不够
4. 扣钱、吐钞票

这就是业务逻辑：按照规则一步步处理
```

### 1.2 项目中的业务逻辑

在这个项目中，业务逻辑包括：

| 业务场景 | 业务逻辑 |
|----------|----------|
| 用户登录 | 验证用户名密码 → 生成令牌 → 返回 |
| 创建任务 | 检查权限 → 验证数据 → 保存 → 计算风险 |
| 分配任务 | 计算匹配分数 → 推荐最合适的人 |
| 更新进度 | 检查权限 → 更新数据 → 重新计算风险 |

### 1.3 Service 层的职责

```
Service 层的职责：

┌─────────────────────────────────────────────────────────┐
│  1. 接收 Controller 传来的参数                           │
│  2. 执行业务规则验证                                     │
│  3. 调用 Mapper 操作数据                                 │
│  4. 调用算法进行计算                                     │
│  5. 返回处理结果                                         │
└─────────────────────────────────────────────────────────┘
```

## 二、AuthService —— 登录服务详解

### 2.1 登录流程图

```
用户输入用户名密码
        ↓
┌─────────────────────────────────────────────────────────┐
│  AuthService.login() 方法                                │
│                                                         │
│  第1步：验证用户名密码                                     │
│  authenticationManager.authenticate()                   │
│  （如果验证失败，抛出异常，登录失败）                        │
│                                                         │
│  第2步：提取用户信息                                       │
│  从认证结果中获取 UserPrincipal                          │
│                                                         │
│  第3步：生成令牌                                         │
│  - 生成访问令牌（accessToken）                            │
│  - 生成刷新令牌（refreshToken）                           │
│                                                         │
│  第4步：返回结果                                         │
│  返回令牌、用户ID、用户名、角色                            │
└─────────────────────────────────────────────────────────┘
        ↓
返回给前端
```

### 2.2 代码详细解析

打开 `service/AuthService.java`：

```java
@Service    // 标记这是一个服务类
@RequiredArgsConstructor    // 自动生成构造函数
public class AuthService {

    private final AuthenticationManager authenticationManager;    // 认证管理器
    private final JwtTokenProvider jwtTokenProvider;    // JWT 工具
    private final UserMapper userMapper;    // 用户数据操作

    public Map<String, Object> login(String username, String password) {
        try {
            // ============ 第1步：验证用户名密码 ============
            Authentication auth = authenticationManager.authenticate(
                new UsernamePasswordAuthenticationToken(username, password)
            );
            // authenticationManager.authenticate() 会：
            // 1. 调用 UserDetailsService 查询用户
            // 2. 比对密码
            // 3. 如果正确，返回认证结果；如果错误，抛出异常
            
            // ============ 第2步：提取用户信息 ============
            UserPrincipal p = (UserPrincipal) auth.getPrincipal();
            // getPrincipal() 获取认证成功的用户信息
            
            // ============ 第3步：生成令牌 ============
            String access = jwtTokenProvider.createAccessToken(
                p.getId(), p.getUsername(), p.getRole()
            );
            String refresh = jwtTokenProvider.createRefreshToken(p.getId());
            
            // ============ 第4步：返回结果 ============
            return Map.of(
                "accessToken", access,
                "refreshToken", refresh,
                "role", p.getRole(),
                "userId", p.getId(),
                "username", p.getUsername()
            );
            
        } catch (Exception e) {
            // 验证失败，抛出统一异常
            throw new ApiException(HttpStatus.UNAUTHORIZED, "用户名或密码错误");
        }
    }
}
```

### 2.3 刷新令牌机制

为什么要两个令牌？

```
访问令牌（Access Token）：
- 有效期短（如 30 分钟）
- 用于日常请求
- 泄露风险低（因为有效期短）

刷新令牌（Refresh Token）：
- 有效期长（如 7 天）
- 用于获取新的访问令牌
- 存储更安全

流程：
1. 用户登录 → 获得 accessToken 和 refreshToken
2. 用 accessToken 访问 API
3. accessToken 过期 → 用 refreshToken 获取新的 accessToken
4. refreshToken 过期 → 需要重新登录
```

刷新令牌的代码：

```java
public Map<String, Object> refresh(String refreshToken) {
    try {
        // 第1步：解析刷新令牌
        var claims = jwtTokenProvider.parseRefreshToken(refreshToken);
        
        // 第2步：检查令牌类型
        if (!"refresh".equals(claims.get("type"))) {
            throw new ApiException(HttpStatus.UNAUTHORIZED, "无效令牌");
        }
        
        // 第3步：获取用户ID
        Long userId = Long.parseLong(claims.getSubject());
        
        // 第4步：检查用户状态（防止已禁用用户刷新令牌）
        var u = userMapper.findById(userId);
        if (u == null || u.getStatus() == 0) {
            throw new ApiException(HttpStatus.UNAUTHORIZED, "用户无效");
        }
        
        // 第5步：签发新令牌
        String access = jwtTokenProvider.createAccessToken(u.getId(), u.getUsername(), u.getRole());
        String refresh = jwtTokenProvider.createRefreshToken(u.getId());
        
        return Map.of("accessToken", access, "refreshToken", refresh);
    } catch (ApiException e) {
        throw e;
    } catch (Exception e) {
        throw new ApiException(HttpStatus.UNAUTHORIZED, "刷新失败");
    }
}
```

## 三、TaskService —— 任务服务详解

### 3.1 创建任务流程

```
管理者创建任务
        ↓
┌─────────────────────────────────────────────────────────┐
│  TaskService.create() 方法                               │
│                                                         │
│  1. 检查团队是否存在                                      │
│  2. 检查用户是否是团队管理者                               │
│  3. 设置默认值（状态、难度、优先级等）                      │
│  4. 检查被指派人是否是团队成员                             │
│  5. 初始化风险等级为绿色                                   │
│  6. 保存任务到数据库                                      │
│  7. 处理任务依赖关系                                      │
│  8. 记录审计日志                                         │
│  9. 计算风险                                             │
└─────────────────────────────────────────────────────────┘
        ↓
返回新创建的任务
```

### 3.2 代码详细解析

```java
@Service
@RequiredArgsConstructor
public class TaskService {

    // 注入各种依赖
    private final TaskMapper taskMapper;
    private final TaskDependencyMapper taskDependencyMapper;
    private final UserMapper userMapper;
    private final TeamService teamService;
    private final AuditService auditService;
    private final AppProperties appProperties;
    private final InAppMessageService inAppMessageService;

    // ========== 创建任务 ==========
    @Transactional    // 开启事务，要么全成功，要么全失败
    public Task create(Long teamId, Task task, List<Long> predecessorIds, UserPrincipal actor) {
        
        // 第1步：检查团队是否存在，并验证权限
        var team = teamService.requireTeam(teamId);
        teamService.assertManager(actor, team);
        // 如果团队不存在或用户不是管理者，会抛出异常
        
        // 第2步：设置任务的基本信息
        task.setTeamId(teamId);
        task.setCreatorId(actor.getId());
        
        // 第3步：设置默认值
        if (task.getStatus() == null) {
            task.setStatus("CREATED");    // 默认状态：已创建
        }
        if (task.getDifficulty() == null) {
            task.setDifficulty(BigDecimal.valueOf(1));    // 默认难度：1
        }
        if (task.getPriority() == null) {
            task.setPriority(3);    // 默认优先级：3（中等）
        }
        if (task.getProgress() == null) {
            task.setProgress(0);    // 默认进度：0%
        }
        
        // 第4步：检查被指派人
        if (task.getAssigneeId() != null) {
            // 被指派人必须是已审核通过的团队成员
            if (!teamService.isApprovedMember(teamId, task.getAssigneeId())) {
                throw new ApiException(HttpStatus.BAD_REQUEST, "被指派人须为已通过审核的成员");
            }
            User assignee = userMapper.findById(task.getAssigneeId());
            if (assignee == null) {
                throw new ApiException(HttpStatus.BAD_REQUEST, "用户不存在");
            }
        }
        
        // 第5步：初始化风险
        task.setRiskLevel("GREEN");    // 默认风险等级：绿色
        task.setDelayProbability(BigDecimal.ZERO);    // 延期概率：0
        
        // 第6步：保存任务
        taskMapper.insert(task);
        Long sid = task.getId();    // 获取自增ID
        
        // 第7步：处理任务依赖关系
        if (predecessorIds != null) {
            for (Long pid : predecessorIds) {
                if (pid == null || pid.equals(sid)) {
                    continue;    // 跳过空值和自己
                }
                Task pred = taskMapper.findById(pid);
                if (pred == null || !pred.getTeamId().equals(teamId)) {
                    throw new ApiException(HttpStatus.BAD_REQUEST, "前置任务无效");
                }
                TaskDependency dep = new TaskDependency();
                dep.setPredecessorId(pid);
                dep.setSuccessorId(sid);
                taskDependencyMapper.insert(dep);
            }
        }
        
        // 第8步：记录审计日志
        auditService.log(actor.getId(), "TASK_CREATE", "Task", sid, task.getTitle());
        
        // 第9步：计算风险
        recomputeRisk(sid);
        
        return taskMapper.findById(sid);
    }
}
```

### 3.3 关键概念解释

**@Transactional（事务）**：

```
事务的作用：保证多个操作要么全成功，要么全失败

没有事务时：
操作1：插入任务 ✅
操作2：插入依赖关系 ❌（失败了）
结果：任务存在但没有依赖关系，数据不一致！

有事务时：
操作1：插入任务 ✅
操作2：插入依赖关系 ❌（失败了）
结果：操作1被撤销，数据库回到之前的状态
```

**乐观锁（version 字段）**：

```java
int n = taskMapper.update(existing);
if (n == 0) {
    throw new ApiException(HttpStatus.CONFLICT, "任务已被他人更新，请刷新后重试");
}
```

乐观锁解决"并发修改"问题：

```
用户A和用户B同时编辑同一个任务：

时间线：
T1: 用户A读取任务（version=1）
T2: 用户B读取任务（version=1）
T3: 用户A保存（成功，version变为2）
T4: 用户B保存（失败！因为version已经变成2了）

乐观锁：检查version是否匹配，不匹配说明有人改过
```

### 3.4 更新进度

```java
@Transactional
public Task updateProgress(Long taskId, int progress, int version, UserPrincipal actor) {
    // 第1步：获取任务
    Task t = requireTask(taskId);
    
    // 第2步：检查权限（只有执行人、管理员、管理者可以更新）
    boolean isAssignee = t.getAssigneeId() != null && t.getAssigneeId().equals(actor.getId());
    if (!isAssignee) {
        if (!"ADMIN".equals(actor.getRole()) && !"MANAGER".equals(actor.getRole())) {
            throw new ApiException(HttpStatus.FORBIDDEN, "只能更新自己的任务进度");
        }
    }
    
    // 第3步：检查任务状态
    if ("COMPLETED".equals(t.getStatus())) {
        throw new ApiException(HttpStatus.BAD_REQUEST, "已完成任务不可修改进度");
    }
    
    // 第4步：检查进度范围（0-89%，100%只能通过审批完成）
    if (progress < 0 || progress > PROGRESS_REPORT_MAX) {
        throw new ApiException(HttpStatus.BAD_REQUEST, 
            "进度仅可上报 0%–89%；任务审批通过为「已完成」后将自动更新为 100%");
    }
    
    // 第5步：更新进度
    int n = taskMapper.updateProgress(taskId, progress, version);
    if (n == 0) {
        throw new ApiException(HttpStatus.CONFLICT, "任务已被他人更新，请刷新后重试");
    }
    
    // 第6步：通知管理者
    if (isAssignee && progress != oldProgress) {
        notifyManagerProgressReport(t, oldProgress, progress, actor);
    }
    
    // 第7步：重新计算风险
    recomputeRisk(taskId);
    
    return taskMapper.findById(taskId);
}
```

### 3.5 风险计算

```java
public void recomputeRisk(Long taskId) {
    // 第1步：获取任务
    Task t = requireTask(taskId);
    String previousLevel = normalizeRiskLevel(t.getRiskLevel());
    
    // 第2步：检查任务状态（只有进行中的任务才计算风险）
    if (!List.of("CREATED", "IN_PROGRESS", "SUSPENDED").contains(t.getStatus())) {
        taskMapper.updateRisk(taskId, "GREEN", BigDecimal.ZERO);
        return;
    }
    
    // 第3步：获取执行人信息
    User assignee = t.getAssigneeId() != null ? userMapper.findById(t.getAssigneeId()) : null;
    double delayHist = assignee != null && assignee.getDelayHistoryScore() != null
            ? assignee.getDelayHistoryScore().doubleValue() : 0;
    
    // 第4步：调用算法计算延期概率
    BigDecimal p = RiskModel.delayProbability(
        appProperties.getRisk(),
        t.getProgress(),
        start,
        t.getDeadline(),
        diff,
        delayHist
    );
    
    // 第5步：确定风险等级
    String level = RiskModel.level(appProperties.getRisk(), p);
    
    // 第6步：更新数据库
    taskMapper.updateRisk(taskId, level, p);
    
    // 第7步：如果风险上升，通知管理者
    if (riskEscalatedAcrossThreshold(previousLevel, level)) {
        notifyManagerRiskEscalation(t, previousLevel, level, p);
    }
}
```

## 四、AssignmentService —— 智能分配服务

### 4.1 智能分配的核心逻辑

```
创建任务后，系统自动推荐最合适的执行人

计算公式：
总分 = 技能匹配分 × 权重1 + 空闲程度 × 权重2 + 历史绩效 × 权重3

技能匹配分：这个人的技能和任务要求匹配程度
空闲程度：这个人现在有多空闲（1 - 当前负载）
历史绩效：这个人过去的工作表现如何
```

### 4.2 代码详细解析

```java
@Service
@RequiredArgsConstructor
public class AssignmentService {

    private final UserMapper userMapper;
    private final TaskMapper taskMapper;
    private final AhpService ahpService;
    private final TeamService teamService;
    private final TeamMemberMapper teamMemberMapper;

    public List<Map<String, Object>> recommend(Long teamId, String requiredSkillsJson, UserPrincipal actor) {
        
        // 第1步：检查权限
        var team = teamService.requireTeam(teamId);
        teamService.assertManager(actor, team);
        
        // 第2步：获取权重配置
        double[] w = ahpService.weightsOrDefault(teamId);
        // w[0] = 技能权重
        // w[1] = 负载权重
        // w[2] = 绩效权重
        
        // 第3步：获取团队成员列表
        List<Long> userIds = teamMemberMapper.findApprovedUserIds(teamId);
        
        // 第4步：遍历每个成员，计算得分
        List<Map<String, Object>> out = new ArrayList<>();
        for (Long uid : userIds) {
            User u = userMapper.findById(uid);
            if (u == null || u.getStatus() == 0) {
                continue;    // 跳过无效用户
            }
            
            // 计算技能匹配分（0-1）
            double s = SkillMatch.cosineSimilarity(requiredSkillsJson, u.getSkillsJson());
            
            // 计算负载比例（0-1）
            BigDecimal rem = taskMapper.sumRemainingHoursByAssignee(uid);
            if (rem == null) {
                rem = BigDecimal.ZERO;
            }
            BigDecimal cap = u.getBaseCapacity() != null ? u.getBaseCapacity() : BigDecimal.valueOf(40);
            double load = rem.divide(cap, 4, RoundingMode.HALF_UP).doubleValue();
            load = Math.min(1.0, Math.max(0, load));    // 限制在 0-1 范围
            
            // 计算绩效分（0-1）
            double perf = u.getAvgPerformance() != null ? u.getAvgPerformance().doubleValue() / 100.0 : 0.75;
            perf = Math.min(1.0, Math.max(0, perf));
            
            // 计算总分
            double score = w[0] * s + w[1] * (1 - load) + w[2] * perf;
            // w[0] * s：技能匹配的贡献
            // w[1] * (1 - load)：空闲程度的贡献（负载越低，空闲越高）
            // w[2] * perf：历史绩效的贡献
            
            // 保存结果
            out.add(Map.of(
                "userId", uid,
                "username", u.getUsername(),
                "displayName", u.getDisplayName(),
                "skillMatch", round4(s),
                "loadRatio", round4(load),
                "performanceNorm", round4(perf),
                "totalScore", round4(score)
            ));
        }
        
        // 第5步：按总分从高到低排序
        out.sort(Comparator.comparingDouble(m -> -((Number) m.get("totalScore")).doubleValue()));
        
        return out;
    }
    
    private static double round4(double v) {
        return Math.round(v * 10000) / 10000.0;
    }
}
```

### 4.3 分数计算示例

假设：
- 技能权重 = 0.5
- 负载权重 = 0.3
- 绩效权重 = 0.2

张三的数据：
- 技能匹配分 = 0.8（技能很匹配）
- 当前负载 = 0.6（还剩 40% 空闲）
- 历史绩效 = 0.85（绩效不错）

张三的总分 = 0.5 × 0.8 + 0.3 × (1 - 0.6) + 0.2 × 0.85
          = 0.5 × 0.8 + 0.3 × 0.4 + 0.2 × 0.85
          = 0.4 + 0.12 + 0.17
          = 0.69

李四的数据：
- 技能匹配分 = 0.5（技能一般）
- 当前负载 = 0.2（还剩 80% 空闲）
- 历史绩效 = 0.9（绩效很好）

李四的总分 = 0.5 × 0.5 + 0.3 × (1 - 0.2) + 0.2 × 0.9
          = 0.5 × 0.5 + 0.3 × 0.8 + 0.2 × 0.9
          = 0.25 + 0.24 + 0.18
          = 0.67

结论：张三稍微优先于李四

### 5.2 全局异常处理器

打开 `exception/GlobalExceptionHandler.java`：

```java
@RestControllerAdvice
public class GlobalExceptionHandler {

    @ExceptionHandler(ApiException.class)
    public ResponseEntity<Map<String, String>> handleApiException(ApiException e) {
        return ResponseEntity.status(e.getStatus())
                .body(Map.of("error", e.getMessage()));
    }
    
    @ExceptionHandler(Exception.class)
    public ResponseEntity<Map<String, String>> handleException(Exception e) {
        return ResponseEntity.status(HttpStatus.INTERNAL_SERVER_ERROR)
                .body(Map.of("error", "服务器内部错误"));
    }
}
```

这样，所有的异常都会被统一处理：

```
抛出异常：
throw new ApiException(HttpStatus.BAD_REQUEST, "用户名已存在");

自动转换为 JSON 响应：
HTTP 400
{
    "error": "用户名已存在"
}
```
