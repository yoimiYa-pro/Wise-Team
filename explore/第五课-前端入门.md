# 第五课：前端入门 —— Vue 3 基础

## 一、什么是前端？

### 1.1 用大白话说

想象你在餐厅吃饭：

- **前端** = 餐厅的大堂、菜单、服务员
  - 你能看到的地方
  - 你能操作的界面
  - 和你互动的部分

- **后端** = 餐厅的厨房
  - 在幕后工作
  - 处理你的订单
  - 你看不到但很重要

### 1.2 前端的三大核心技术

```
┌─────────────────────────────────────────────────────────┐
│  HTML（结构）                                             │
│  就像房子的骨架：墙、门、窗                                 │
│  决定网页有什么内容                                        │
└─────────────────────────────────────────────────────────┘

┌─────────────────────────────────────────────────────────┐
│  CSS（样式）                                              │
│  就像房子的装修：颜色、大小、位置                           │
│  决定网页长什么样                                         │
└─────────────────────────────────────────────────────────┘

┌─────────────────────────────────────────────────────────┐
│  JavaScript（行为）                                       │
│  就像房子的电器：灯光、空调、门禁                           │
│  决定网页能做什么                                         │
└─────────────────────────────────────────────────────────┘
```

### 1.3 为什么用 Vue？

传统的开发方式：
```html
<!-- 需要手动操作 DOM，很繁琐 -->
<div id="app"></div>
<script>
  document.getElementById('app').innerHTML = 'Hello';
</script>
```

用 Vue 开发：
```html
<!-- 声明式，只需要关心数据 -->
<div id="app">{{ message }}</div>
<script>
  const app = {
    data() {
      return { message: 'Hello' }
    }
  }
</script>
```

Vue 的优势：
- **声明式**：你只需要关心"数据是什么"，不用管"怎么更新界面"
- **组件化**：把界面拆成小块，便于复用和维护
- **生态丰富**：有很多现成的组件库可以用

## 二、Vue 3 的核心概念

### 2.1 组件（Component）

组件就是**"可复用的界面块"**：

```
一个网页可以拆分成多个组件：

┌─────────────────────────────────────────────────────────┐
│  页面（Page）                                             │
│  ┌─────────────────────────────────────────────────────┐│
│  │  导航栏组件（Header）                                 ││
│  └─────────────────────────────────────────────────────┘│
│  ┌──────────────┐  ┌──────────────┐  ┌──────────────┐  │
│  │ 侧边栏组件    │  │  内容区域     │  │  右侧面板    │  │
│  │ (Sidebar)    │  │ (Content)    │  │ (Panel)     │  │
│  │              │  │              │  │              │  │
│  │ ┌──────────┐ │  │ ┌──────────┐ │  │              │  │
│  │ │菜单项组件 │ │  │ │任务卡片  │ │  │              │  │
│  │ └──────────┘ │  │ └──────────┘ │  │              │  │
│  └──────────────┘  └──────────────┘  └──────────────┘  │
└─────────────────────────────────────────────────────────┘
```

### 2.2 单文件组件（.vue 文件）

每个 `.vue` 文件包含三部分：

```vue
<!-- 模板：HTML 结构 -->
<template>
  <div class="my-component">
    <h1>{{ title }}</h1>
    <button @click="count++">点击了 {{ count }} 次</button>
  </div>
</template>

<!-- 脚本：JavaScript 逻辑 -->
<script setup lang="ts">
import { ref } from 'vue';

const title = ref('我的组件');
const count = ref(0);
</script>

<!-- 样式：CSS -->
<style scoped>
.my-component {
  padding: 20px;
  border: 1px solid #ccc;
}
</style>
```

### 2.3 响应式数据

响应式数据就是**"会自动更新界面的数据"**：

```javascript
import { ref } from 'vue';

// 创建响应式数据
const count = ref(0);

// 当 count 改变时，界面会自动更新
count.value = 1;  // 界面自动变成 1
```

对比普通变量：
```javascript
// 普通变量
let count = 0;
count = 1;  // 界面不会更新！需要手动操作 DOM
```

## 四、路由（Router）

### 4.1 什么是路由？

路由就是**"根据 URL 显示不同页面"**：

```
用户访问 /login → 显示登录页面
用户访问 /dashboard → 显示看板页面
用户访问 /tasks/123 → 显示 ID 为 123 的团队任务页面
```

### 4.2 路由配置解析

打开 `router/index.ts`：

```typescript
import { createRouter, createWebHistory } from "vue-router";

const router = createRouter({
  history: createWebHistory(),  // 使用 HTML5 History 模式
  routes: [
    // 登录页面（公开访问）
    {
      path: "/login",
      name: "login",
      component: () => import("../views/LoginView.vue"),
      meta: { public: true },  // 标记为公开，不需要登录
    },
    
    // 主布局（需要登录）
    {
      path: "/",
      component: () => import("../layouts/MainLayout.vue"),
      meta: { requiresAuth: true },  // 标记为需要登录
      children: [  // 子路由
        { path: "", redirect: "/dashboard" },  // 默认重定向
        { path: "dashboard", name: "dashboard", component: () => import("../views/DashboardView.vue") },
        { path: "teams", name: "teams", component: () => import("../views/TeamsView.vue") },
        { path: "tasks/:teamId", name: "tasks", component: () => import("../views/TasksView.vue") },
        // ... 更多子路由
      ],
    },
  ],
});
```

### 4.3 路由守卫

```typescript
router.beforeEach((to, _from, next) => {
  // 如果是公开页面，直接放行
  if (to.meta.public) {
    next();
    return;
  }
  
  // 如果需要登录但没有 token，跳转到登录页
  if (to.meta.requiresAuth && !localStorage.getItem("accessToken")) {
    next({ name: "login", query: { redirect: to.fullPath } });
    return;
  }
  
  // 其他情况放行
  next();
});
```

路由守卫的作用：

```
访问流程：

用户访问 /dashboard
      ↓
路由守卫检查：是否需要登录？是
      ↓
检查：有 token 吗？没有
      ↓
重定向到 /login?redirect=/dashboard
      ↓
用户登录成功后，跳转回 /dashboard
```

## 五、组件详解

### 5.1 登录页面（LoginView.vue）

```vue
<template>
  <div class="login-wrap">
    <!-- Ant Design 的卡片组件 -->
    <a-card title="团队任务管理与绩效评估" style="width: 400px">
      <!-- 表单 -->
      <a-form layout="vertical" :model="form" @finish="onFinish">
        <!-- 用户名输入框 -->
        <a-form-item
          label="用户名"
          name="username"
          :rules="[{ required: true, message: '请输入用户名' }]"
        >
          <a-input v-model:value="form.username" placeholder="admin" />
        </a-form-item>
        
        <!-- 密码输入框 -->
        <a-form-item
          label="密码"
          name="password"
          :rules="[{ required: true, message: '请输入密码' }]"
        >
          <a-input-password v-model:value="form.password" placeholder="密码" />
        </a-form-item>
        
        <!-- 登录按钮 -->
        <a-button type="primary" html-type="submit" block>登录</a-button>
      </a-form>
    </a-card>
  </div>
</template>

<script setup lang="ts">
import { onMounted, reactive } from "vue";
import { useRouter } from "vue-router";
import { message } from "ant-design-vue";
import client from "../api/client";

const router = useRouter();

// 响应式表单数据
const form = reactive({ username: "", password: "" });

// 组件挂载时清除旧的登录信息
onMounted(() => {
  localStorage.removeItem("accessToken");
  localStorage.removeItem("refreshToken");
});

// 表单提交处理
async function onFinish() {
  try {
    // 发送登录请求
    const { data } = await client.post("/auth/login", form);
    
    // 保存登录信息
    localStorage.setItem("accessToken", data.accessToken);
    localStorage.setItem("refreshToken", data.refreshToken);
    localStorage.setItem("role", data.role);
    localStorage.setItem("userId", String(data.userId));
    localStorage.setItem("username", data.username);
    
    // 显示成功消息
    message.success("登录成功");
    
    // 跳转到首页
    router.push("/");
  } catch {
    message.error("登录失败");
  }
}
</script>

<style scoped>
.login-wrap {
  min-height: 100vh;
  display: flex;
  align-items: center;
  justify-content: center;
  background: linear-gradient(135deg, #1a1a2e, #16213e);
}
</style>
```

### 5.2 关键语法解释

**v-model（双向绑定）**：
```vue
<a-input v-model:value="form.username" />
```
等价于：
```vue
<a-input 
  :value="form.username" 
  @input="form.username = $event.target.value" 
/>
```
用大白话说：输入框的值和 `form.username` 双向同步，改一个另一个也变。

**@finish（事件绑定）**：
```vue
<a-form @finish="onFinish">
```
当表单验证通过并提交时，调用 `onFinish` 函数。

**reactive（响应式对象）**：
```typescript
const form = reactive({ username: "", password: "" });
```
创建一个响应式对象，当 `form.username` 或 `form.password` 改变时，界面自动更新。

### 5.3 主布局（MainLayout.vue）

主布局包含三个部分：

```
┌─────────────────────────────────────────────────────────┐
│  侧边栏（Sider）                                          │
│  ┌─────────────┐  ┌────────────────────────────────────┐│
│  │ Logo        │  │  顶部栏（Header）                    ││
│  │             │  │  消息 | 用户名 | 退出                ││
│  │ 菜单项1     │  ├────────────────────────────────────┤│
│  │ 菜单项2     │  │                                    ││
│  │ 菜单项3     │  │  内容区（Content）                  ││
│  │ ...         │  │  <router-view />                   ││
│  │             │  │  显示当前页面的内容                  ││
│  │             │  │                                    ││
│  └─────────────┘  └────────────────────────────────────┘│
└─────────────────────────────────────────────────────────┘
```

关键代码片段：

```vue
<template>
  <a-layout class="app-root-layout">
    <!-- 侧边栏 -->
    <a-layout-sider class="app-sider" breakpoint="lg" :collapsed-width="64">
      <div class="logo">任务管理与绩效评估系统</div>
      
      <!-- 菜单 -->
      <a-menu v-model:selectedKeys="selectedKeys" theme="dark" mode="inline" @click="onMenuClick">
        <a-menu-item v-for="item in menuItems" :key="item.key">
          <template #icon><component :is="item.icon" /></template>
          {{ item.label }}
        </a-menu-item>
      </a-menu>
    </a-layout-sider>
    
    <!-- 主内容区 -->
    <a-layout class="app-main">
      <!-- 顶部栏 -->
      <a-layout-header class="header">
        <span class="user-meta">{{ username }}（{{ roleDisplay }}）</span>
        <a href="#" @click.prevent="logout">退出</a>
      </a-layout-header>
      
      <!-- 内容区 -->
      <a-layout-content class="content">
        <router-view />  <!-- 子页面显示在这里 -->
      </a-layout-content>
    </a-layout>
  </a-layout>
</template>

<script setup lang="ts">
import { computed, ref } from "vue";
import { useRouter } from "vue-router";

const router = useRouter();

// 计算属性：从 localStorage 获取角色
const roleDisplay = computed(() => localStorage.getItem("role") || "MEMBER");
const username = computed(() => localStorage.getItem("username") || "");

// 根据角色动态生成菜单
const menuItems = computed(() => {
  const r = roleDisplay.value;
  const items = [];
  
  if (r === "ADMIN") {
    items.push(
      { key: "/admin/users", label: "用户管理", icon: UserOutlined },
      { key: "/admin/audit", label: "审计日志", icon: AuditOutlined },
    );
  }
  
  if (r === "MANAGER" || r === "ADMIN") {
    items.push(
      { key: "/dashboard", label: "管理者看板", icon: DashboardOutlined },
      { key: "/teams", label: "团队管理", icon: TeamOutlined },
    );
  }
  
  // ... 更多菜单项
  
  return items;
});

// 退出登录
function logout() {
  localStorage.clear();
  router.push("/login");
}
</script>
```

### 5.4 computed（计算属性）

```typescript
const roleDisplay = computed(() => localStorage.getItem("role") || "MEMBER");
```

计算属性的特点：
- 有缓存：只有依赖变化时才重新计算
- 自动更新：依赖变化时，使用它的地方自动更新

对比普通函数：
```typescript
// 每次调用都执行
function getRole() {
  return localStorage.getItem("role") || "MEMBER";
}

// 只有 role 变化时才重新计算
const roleDisplay = computed(() => localStorage.getItem("role") || "MEMBER");
```

## 六、Ant Design Vue 组件库

### 6.1 什么是组件库？

组件库就是**"现成的 UI 零件"**：
- 按钮、输入框、表格、弹窗...
- 不用自己写，拿来就用
- 样式统一，交互规范

### 6.2 常用组件

**按钮（Button）**：
```vue
<a-button type="primary">主要按钮</a-button>
<a-button>默认按钮</a-button>
<a-button type="dashed">虚线按钮</a-button>
<a-button danger>危险按钮</a-button>
```

**输入框（Input）**：
```vue
<a-input v-model:value="text" placeholder="请输入" />
<a-input-password v-model:value="password" placeholder="请输入密码" />
<a-textarea v-model:value="content" placeholder="多行文本" />
```

**表格（Table）**：
```vue
<a-table :columns="columns" :data-source="data" />
```

**表单（Form）**：
```vue
<a-form :model="form" @finish="onSubmit">
  <a-form-item label="名称" name="name" :rules="[{ required: true }]">
    <a-input v-model:value="form.name" />
  </a-form-item>
  <a-button type="primary" html-type="submit">提交</a-button>
</a-form>
```

**消息提示（Message）**：
```typescript
import { message } from 'ant-design-vue';

message.success('操作成功');
message.error('操作失败');
message.warning('警告信息');
```

## 七、生命周期钩子

### 7.1 什么是生命周期？

组件从创建到销毁的过程：

```
创建 → 挂载 → 更新 → 卸载
```

### 7.2 常用钩子

```typescript
import { onMounted, onUnmounted, ref } from 'vue';

const data = ref(null);

// 组件挂载后执行（类似 DOM 加载完成）
onMounted(() => {
  console.log('组件已挂载');
  fetchData();  // 通常在这里获取数据
});

// 组件卸载前执行（清理工作）
onUnmounted(() => {
  console.log('组件将卸载');
  clearInterval(timer);  // 清理定时器
});
```

### 7.3 实际应用

```typescript
// 轮询获取未读消息数
let msgPoll = null;

onMounted(() => {
  fetchUnread();  // 立即获取一次
  msgPoll = setInterval(fetchUnread, 60000);  // 每分钟获取一次
});

onUnmounted(() => {
  if (msgPoll) clearInterval(msgPoll);  // 组件销毁时清理定时器
});
```
